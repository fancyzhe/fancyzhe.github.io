雅虎35军规（1） -- 内容和服务（译）

发表于 2018-12-28

原文地址雅虎35军规

雅虎的专业性能团队为了web网页能更加快速，制定了7个方面的35条最好实践方式。

分成这个七个方面：内容、服务、缓存、css样式、JavaScript、图像、移动端、全部

内容

  1. 减少http请求次数

80%最终相应的时间是花费在前端界面上，而时间的大部分是花费在下载页面上所有需要的部分，比如图像、css、script、flash等。通过减少组成部分的数量去减少http请求去渲染页面，这是为了页面渲染更快的关键。

在页面上减少组成部分的数量也是简化页面的设计一种方式，而这种方式去构建更多更丰富的内容怎么样才能有更快的相应时间呢？当页面有更多的设计，减少http的请求次数也是一些技巧。

组合文件 是一种通过将所有的脚本组合到单个脚本中，同样的将css样式表到单个的css文件中去减少http请求的次数。组合文件是不同页面的css和脚本的更大挑战，但是做这部分提高了响应时间。

css sprites（雪碧图） 是减少图片请求的首选方式，用一张图片组合你的背景图配合css的background-image和background-position去显示你想要的图片效果。

图像映射 组合多张图片到一个图片中，总的大小是相同的，但是减少了http请求加快了页面加载。图像映射只能在相近的页面上运行，比如导航栏。定义图像映射的坐标可能出错，在导航栏使用图像映射也不太可行，所以这个是不太推荐的。

使用URL的形式内嵌图像到真实的页面上，这也可以减少你的HTML内容，组合内联图片到你的css文件中是减少http请求和你页面大小的一种方式。内联图片目前不被所有的主流浏览器支持。

在你的页面中减少http请求是在刚开始加载的时候，对于第一次访问网站的用户来说这是最重要的性能参考依据。正如Tenni Theurer的博客中提到，

为了让没有缓存访问网站的40%-60%的日常用户有更好的用户体验，让你的网站第一次访问的更快是最关键的。

服务

  1. 使用分布式网络（CDN）

用户跟你web服务器的距离影响着相应的时间，通过多点发布你的内容，站在用户的角度多个地方的服务器会让你的页面加载的更快。但是你应该从哪里开始呢？

作为多地点分布式内容的第一步，不要为了在分布式架构上运行而尝试重新设计你的web应用。依赖应用，改变你的架构可能有很多复杂的任务比如同步会话状态和通过本地服务器复制数据库传输。尝试着去缩小用户和你的内容的距离可能被延迟或者没有通过这个架构的步骤。

应该牢记的是80%-90%的相应时间都会被花费在下载所有页面的图像、css、JavaScript、flash等。这是性能黄金法则（Performance Golden Rule）。相比开始重新设计你的程序架构这样的困难任务，更好的方式是将你的静态内容分布式。这不仅仅让响应时间大大减少，并且对于分布式网络更容易实现。

CDN是分布式网络给用户高效的发送内容通过多地点传输内容。这个服务器选择传输内容给专门的用户是建立在以距离测量的更近的网络。举一个例子，最少的网络ping值服务器和最快响应服务器是更合适的。

一些大型的网络公司有自己的CDN服务，但是使用CDN服务提供者是有一定的成本效益的。比如Akamai Technologies、EdgeCast、level3。对于创业公司（start-up companies）和私人站点，CDN服务费用过高，但是为了让你的目标群众更多更全球化，CDN是减少响应时间的必要方式。在雅虎，网站静态内容放在CDN上让性能提高和响应时间减少20%甚至更多。转变为CDN是一种相对改变代码来说更简单的方式去显著改变你网站的加载速度。

关键词：Minimie HTTP Requests,Content Delivery NetWork
